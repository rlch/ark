# ark

Zellij-native agent orchestration.

Spawns AI coding agents into dedicated zellij sessions with live diff, git status, and progress panes. Pluggable via two-layer adapter model:

- **Engines** extract structured signal from agent CLIs (`ClaudeCodeEngine` injects hooks, tails transcripts)
- **Orchestrators** encode workflow methodology (`CavekitOrchestrator`, `ClaudeCodeOrchestrator`)

Status: early development. See `context/kits/` for the spec, `context/plans/` for the build site.

## Installation

Once a `v*` tag is pushed, `.github/workflows/release.yml` produces prebuilt
tarballs (`aarch64`/`x86_64` on macOS + Linux) and attaches them to the
GitHub Release. The install paths below then work against that release.

### Homebrew (macOS + Linuxbrew)

```sh
brew tap rlch/ark
brew install ark
# or in one go
brew install rlch/ark/ark
```

The tap repo is `rlch/homebrew-ark` (auto-updated by cargo-dist on release —
see "Release maintainer bootstrap" below for the one-time setup).

### cargo install (any platform with a Rust toolchain)

```sh
cargo install ark-cli
```

Builds `ark` from source. Slow but works on any target with a working
compiler.

### cargo binstall (fast — prebuilt binaries)

```sh
cargo binstall ark-cli
```

Uses the `[package.metadata.binstall]` block in `crates/cli/Cargo.toml` to
fetch the prebuilt tarball for your host triple from the GitHub Release.
Falls back to `cargo install` if no prebuilt matches.

### Manual download

See [github.com/rlch/ark/releases](https://github.com/rlch/ark/releases)
for tarballs (`ark-<version>-<target>.tar.xz`) and their SHA256 sums.
Extract and drop `ark` + `ark-hook` into your `PATH`.

Each release also publishes the standalone wasm plugins as separate
assets for users who only want the plugin binaries (e.g. to load them
into a zellij config without installing `ark` itself):

- `ark-status-v<version>.wasm` + `ark-status-v<version>.wasm.sha256`
- `ark-picker-v<version>.wasm` + `ark-picker-v<version>.wasm.sha256`

Fetch them directly with:

```sh
V=<version>
curl -LO "https://github.com/rlch/ark/releases/download/v${V}/ark-status-v${V}.wasm"
curl -LO "https://github.com/rlch/ark/releases/download/v${V}/ark-status-v${V}.wasm.sha256"
sha256sum -c "ark-status-v${V}.wasm.sha256"
```

### First-run setup

After installing, run:

```sh
ark doctor --fix
```

It checks for `zellij`, `delta`, and the wasm plugins, and offers to
install anything missing — see `context/kits/cavekit-distribution.md`
for the exact flow.

## Release maintainer bootstrap (one-time)

The homebrew formula is auto-generated by cargo-dist, but the tap repo +
GitHub token must exist before the first release. One-time setup:

1. Create an empty public GitHub repo at `rlch/homebrew-ark`.
2. Create a fine-grained PAT with `contents:write` on that tap repo.
3. Add it as `HOMEBREW_TAP_TOKEN` in this repo's Actions secrets.
4. Push a `vX.Y.Z` tag — cargo-dist builds the tarballs (see
   `.github/workflows/release.yml` from T-133) and writes
   `Formula/ark.rb` to the tap repo.

Until step 1-3 are done, tag pushes still produce the GitHub Release +
tarballs; only the homebrew formula push is skipped.
