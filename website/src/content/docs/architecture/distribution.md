Ark ships as a Rust monorepo with two native binaries (`ark`, `ark-hook`), two embedded wasm plugins, and a bundled zellij binary. This page covers the workspace layout, release automation, wasm embedding, zellij pinning, and install flow.

## Cargo workspace

The monorepo has a root `Cargo.toml` with `[workspace.package]` common fields (version, license = MIT, edition = 2024). Crates live under `crates/`:

| Crate | Kind | Output |
|---|---|---|
| `ark-cli` | binary | `ark` |
| `ark-hook` | binary | `ark-hook` |
| `ark-core` | library | supervisor, event bus, state dir, traits |
| `ark-types` | library | AgentId, AgentEvent, Outcome, etc. |
| `ark-config` | library | figment-layered config |
| `ark-mux-zellij` | library | ZellijMux |
| `ark-engines-claude-code` | library | ClaudeCodeEngine |
| `ark-orchestrators-cavekit` | library | CavekitOrchestrator |
| `ark-orchestrators-claude-code` | library | ClaudeCodeOrchestrator |
| `ark-scene` | library | scene compiler, reload, watcher |
| `ark-scene-v2-archive` | library (archived) | archived v2 scene implementation |
| `ark-ext-proto` | library | extension wire protocol |
| `ark-ext-metadata-types` | library | extension wasm metadata types |
| `ark-ext-metadata` | library | extension wasm metadata impl |
| `ark-ext-derive` | proc-macro | extension derive macros |
| `ark-plugin-status` | cdylib (wasm) | `ark-status.wasm` |
| `ark-plugin-picker` | cdylib (wasm) | `ark-picker.wasm` |
| `ark-plugin-ark-bus` | cdylib (wasm) | `ark-bus.wasm` |
| `ark-test-fixtures` | library (dev) | test helpers + path constants |

The `ark pane` subcommand routes into `ark-pane` logic inside `ark-cli` -- not a separate binary. This avoids PATH pollution and extra install burden.

Wasm crates build separately via:

```bash
cargo build --target wasm32-wasip1 --release -p ark-plugin-status
cargo build --target wasm32-wasip1 --release -p ark-plugin-picker
cargo build --target wasm32-wasip1 --release -p ark-plugin-ark-bus
```

## Wasm plugin embedding

The `ark-cli` crate has a `build.rs` that:

1. Compiles all three wasm plugins for `wasm32-wasip1`.
2. Runs `wasm-opt -Oz --enable-bulk-memory` on the output (saves ~15-20%).
3. Copies the resulting `.wasm` files to `$OUT_DIR/wasm/`.
4. Embeds them into the binary via `include_bytes!`.

At runtime, `ark doctor --fix` extracts these bytes to `~/.config/zellij/plugins/ark-{status,picker,bus}.wasm`.

### Size posture

There is no hard byte budget. The zellij-tile floor is ~500 KB raw; comparable plugins land 1-2 MB. Instead of a fixed limit, CI runs a regression check that fails the PR if either plugin grows more than 25% vs main. Absolute sizes are tracked in release notes.

### Size reduction stack

Both wasm crates use these build defaults:

```toml
[profile.release]
opt-level = "z"
lto = "fat"
codegen-units = 1
strip = true
panic = "abort"
```

Additional rules:
- `default-features = false` on every dependency that supports it.
- Avoid `serde_json`, `chrono`, `regex`, `url`, `uuid`, `humantime` in plugins (collectively ~200-400 KB).
- For picker fuzzy matching: use `nucleo-matcher` (leaf deps: `memchr` only), not `fuzzy-matcher` or full `nucleo`.
- Post-process with `wasm-opt -Oz --enable-bulk-memory`.

## Bundled zellij

Ark ships its own zellij binary to control the exact version. Scene compilation depends on specific zellij KDL features (suppressed-pane API, `MessagePlugin` action, merge semantics) that shift between releases.

### How it works

- The prebuilt zellij binary is bundled into each cargo-dist artifact at `share/ark/bin/zellij`.
- The version pin is tracked in `Cargo.toml` at workspace level.
- Ark always launches the bundled binary by absolute path -- no PATH lookup. This prevents accidental version skew.
- `ARK_USE_SYSTEM_ZELLIJ=1` overrides to the system `zellij` from PATH. This is for developer iteration only, not a supported end-user workflow.
- `ark doctor` reports the bundled zellij version and warns if user config uses features newer than the bundled version.

### Why bundle

Zellij's KDL layout language and plugin API evolve between releases. A version mismatch between ark and zellij can cause silent layout failures or missing plugin capabilities. Bundling eliminates this class of bug entirely.

## Release automation (cargo-dist)

Releases are built from git tags via `cargo-dist`:

1. Push a `vX.Y.Z` tag.
2. GitHub Actions runs the release workflow generated by `cargo dist init`.
3. Prebuilt tarballs are uploaded to GitHub Releases for each target.
4. The Homebrew formula auto-updates via cargo-dist's brew integration.

### Targets

| Target | Notes |
|---|---|
| `aarch64-apple-darwin` | Apple Silicon Mac |
| `x86_64-apple-darwin` | Intel Mac |
| `x86_64-unknown-linux-gnu` | Linux x64 |
| `aarch64-unknown-linux-gnu` | Linux arm64 |

### Artifacts per release

- Prebuilt tarball for each target containing `ark`, `ark-hook`, bundled zellij, and embedded wasm plugins.
- SHA256 checksums (cargo-dist default).
- Standalone `.wasm` files for manual plugin installation.

## Install paths

### Homebrew (primary)

```bash
brew install rlch/ark/ark
```

The Homebrew formula is maintained in a `homebrew-ark` tap repo, auto-updated by cargo-dist on each release. The formula depends on `zellij` as well (the bundled copy takes precedence at runtime, but system zellij is useful for fallback paths under `ARK_USE_SYSTEM_ZELLIJ=1`).

### cargo install

```bash
cargo install ark-cli
```

Builds from source. The crate name is `ark-cli` (the `ark` name may collide on crates.io).

### cargo binstall

```bash
cargo binstall ark-cli
```

Works automatically via cargo-dist artifacts -- no extra configuration needed. Downloads the prebuilt binary instead of compiling from source.

### Linux

AUR packages are community-driven. The cargo-dist tarballs are the primary install path for Linux.

No Windows package manager is planned. Windows is not a v1 target.

## First-run experience

After installation, `ark doctor --fix` handles first-run setup:

```
$ ark doctor --fix

ark doctor
  ✓ zellij 0.44.2 (bundled, required ≥ 0.44.1)
  ✓ delta 0.18.2
  ✓ claude on PATH
  ✗ ~/.config/zellij/plugins/ark-status.wasm missing
    install? [y/N] y
    wrote ~/.config/zellij/plugins/ark-status.wasm (420 KB)
  ✗ ~/.config/zellij/plugins/ark-picker.wasm missing
    install? [y/N] y
    wrote ~/.config/zellij/plugins/ark-picker.wasm (712 KB)
  ⚠ ~/.config/ark/config.toml missing (defaults apply)
    create template? [y/N] y
    wrote ~/.config/ark/config.toml
```

`ark doctor --fix` then prints next steps:

1. Add the status bar plugin to zellij's `default_tab_template`.
2. Add a keybind for the picker (recommended `Ctrl+g a`).
3. Try `ark` (bare) or `ark --scene <name>` from a project directory.

## ACP crate pin

Ark pins a specific version of the `agent-client-protocol` Rust crate. The v0.1.0 release ships `agent-client-protocol = "=0.10.4"` (equality pin).

- Release notes cite the bundled ACP crate version and supported `protocolVersion` range.
- `ark doctor` prints the bundled ACP version in its diagnostic header.
- Unstable ACP methods (`session/fork`, `nes/*`, `elicitation/*`, `document/did*`) are not exposed in v1. They are gated behind capability negotiation.

## Release checklist

1. Update version in `Cargo.toml` workspace (all crates share the version).
2. Update `CHANGELOG.md`.
3. Tag `vX.Y.Z` and push.
4. cargo-dist builds + uploads tarballs, checksums, and wasm artifacts.
5. Homebrew formula auto-updates in the tap repo.
6. Verify `brew install rlch/ark/ark` installs the new version.
7. Verify `cargo binstall ark-cli` downloads the correct binary.
