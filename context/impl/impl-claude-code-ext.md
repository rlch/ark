## Tier 2 notes (2026-04-18)

- Serial on main tree per memory. Scope strict: `extensions/claude-code/` + ledgers. NOT touching supervisor / scene / cli / ark-view.
- T-009 was effectively DONE by T-006 (R2 NDJSON-POST-and-exit-0 already lived inside cc-hook). Tier 2 is the ark-side half of the bridge.
- T-010 DIVERGENCE from plan: plan text had "`--first-post` set by settings.json installer on SessionStart entry only". Replaced with sentinel-file-based auto-detection per prompt — the installer doesn't have to track which hook is "first", cc-hook probes `<socket-parent>/cc-hook.handshake` itself. `--first-post` flag kept as manual override (testing, settings.json templates that want to force handshake). Sentinel lives next to `cc-hook.sock` under the session dir so ark's cleanup path removes it for free at session end.
- T-011 minimal-surface choice: accept loop is single-listener-single-sink (no conn pool). `FnMut(SocketEvent) + Send` sink rather than async-sink — callers that want channel forwarding use `tx.send(ev)` in the closure. `SocketEvent` enum carries both `HookFired` and `BridgeVersionMismatch` frames so T-012 doesn't need a parallel channel.
- T-011 `on_session_start` override design choice: trait surface has no event-publisher handle yet (T-014 is the bus-forwarder tier), so the spawned accept_loop uses a log-only sink for now. Failure branches (spec decode, StateLayout::from_env, socket bind) ALL return `Ok(default)` so supervisor's session-launch path isn't blocked on an observability-only extension. Spawned tokio task aborts automatically when runtime tears down; explicit cancellation tracking is T-041 territory.
- T-012 ext_state API: `ark-ext-claude-code` does NOT depend on `ark-core` (status_writer lives there) — adding that dep to write to `status.json::ext_state` would pull the whole core into the extension. Chose on-disk sentinel path explicitly blessed by the prompt fallback. Sentinel path: `$STATE/sessions/<sid>/claude-code.bridge_version_mismatch.json`, shape `{observed, expected, first_seen_at}`. `ark doctor` (T-042) reads with plain `serde_json::from_slice`, no crate dep needed on claude-code side. Atomic via tmp-file + rename pattern mirroring `write_session_status_atomic`.
