- Serial on main tree per memory. Scope strict: `extensions/claude-code/` + ledgers. NO touches to ark-view / scene / scene-macros / supervisor / cli / ark-ext-proto. Head commit at wave start: `4dfd2e7` (Tier 6a landing). All Tier 6b work sits in `src/view.rs` + `src/lib.rs`.
- T-035 CHAR-WISE TRUNCATION: chose `s.chars().count()` / `.chars().take(n)` over byte-slicing because the title format embeds the multibyte `Â·` separator AND agent-type names may include non-ASCII characters a user authors. Byte-slicing at `max_bytes` would risk panicking on a UTF-8 boundary. Pinned by `title_format_handles_multibyte_agent_type_char_wise` â€” compile-time witness that the resulting string is valid UTF-8 after truncation. Ellipsis `"â€¦"` added when content was dropped (common CLI/TUI convention); kit pins "truncated to N chars" which we interpret inclusively â€” 16 content chars + 1 ellipsis = 17 chars wall-clock when truncation fired. `last_tool=None` renders as `"-"` literal rather than an empty segment so the structure of the title stays visually predictable.
- T-035 EMISSION DRIVER SPLIT: title formatting (pure `format_subagent_title`) is in `view.rs`; state + event dispatch + emission construction (`SubagentRegistry::on_ext_event â†’ Option<RenamePaneEmission>`) is in `lib.rs`. Rationale â€” view is a renderer contract (what does the title LOOK like), registry is a reactive observer (when does the title CHANGE). Keeps the pure functions testable without the event/state plumbing overhead, and lets the same title formatter be reused by doctor or list-column tasks without pulling in the registry.
- T-036 SCOPE TRADE-OFF: kit R6 specifies "ratatui-backed tail"; the `pane_emit` RPC surface carries JSON, NOT a ratatui `Frame`. Chose a MINIMAL plain-text formatter over a full ratatui `TextArea` widget. Rationale: (1) the host-side stack renderer is the layer that knows how to composite a ratatui frame into zellij's pane content area â€” that layer doesn't exist in the extension crate; (2) emitting formatted text lines through `pane/emit` is the shape the host can stream natively today; (3) switching to a ratatui widget would require a pipe-style surface (analogue of `Pane<V: ZellijView>::pipe`) the v0.1 `pane_emit` does not expose. Minimal formatter recognises three JSONL shapes (assistant message text, nested tool_use, top-level tool_use) with fallback to raw-line prefix so scene authors still see output when Claude introduces new shapes. Full ratatui hookup flagged as Tier-8 / T-048 (real-zellij PTY gate) concern â€” that test can wire ratatui at the supervisor/stack-renderer layer without revisiting the extension's surface. `ratatui` / `crossterm` deps NOT added to this crate this tier (already in workspace deps if later wiring needs them).
- T-036 TRUNCATION/ROTATION: reused T-025's `TailCursor` directly via `crate::transcript::TailCursor` â€” the cursor already handles length-shrink + inode-rotation (see `poll_new_lines` implementation). `render_transcript_tail` is a thin composition: `cursor.poll_new_lines()?` â†’ `format_transcript_lines(lines, tail_lines)`. Tail-lines bound is enforced BY the formatter, not by the cursor â€” the cursor returns every new line since last poll, and the formatter keeps only the tail window. That ordering is correct because bounding inside the cursor would drop lines the CALLER might want to persist independently (e.g. for cost/token accounting in T-044's list columns).
- T-037 EVENT FILTERING: registry filters BY `event.ext == "claude-code"` first, then by `event.kind âˆˆ {subagent.start, subagent.stop, pre-tool-use}`. Main-session `PreToolUse` (no `agent_id` in payload) is dropped SILENTLY with `None` â€” it's not a bug, that event's tool-use metadata belongs to the main claude pane, not a subagent tile. Unknown `agent_id` (e.g. `pre-tool-use` arriving before its `subagent.start` due to NDJSON reordering â€” shouldn't happen but defensive) also drops; a subsequent `start` will seed the state, and the next `pre-tool-use` will update it. `subagent.stop` WITHOUT a prior `subagent.start` defensively synthesises a `SubagentState` with `agent_type = "subagent"` + the terminal status so a mid-session mount still renders â€” otherwise a mount between `start` and `stop` would render no title at all. Pinned by `registry_subagent_stop_without_start_synthesises_state`.
- T-037 CONCURRENCY: `SubagentRegistry` wraps `HashMap<String, SubagentState>` in `Arc<Mutex<...>>`. Not a hot path â€” subagent hook events fire at human-speed (< O(tens) per minute per live claude-code pane). `DashMap` was considered but rejected: the mutex-based impl is trivially auditable + the kit constraint says "Mutex<HashMap> or DashMap; not a hot path". Poisoning handling: every lock site uses `.lock().ok()` fallbacks that return `None` rather than propagating â€” poisoning = a panic inside the registry itself (shouldn't happen â€” the inside is a `HashMap` insert/get), and silently skipping an emission is strictly better than cascading the panic through the socket-reader task.
