- T-019 SHAPE DECISIONS: (a) `SettingsFile` owns a `serde_json::Value` rather than a typed struct so unknown keys round-trip for free. (b) ark-managed marker chosen as `"ark_managed": true` AT ENTRY LEVEL (not as a comment — Claude Code's JSON parser doesn't preserve comments). `is_ark_managed` is bool-strict: only the exact literal `true` value qualifies, so user-authored "ark_managed": "true" strings or future marker variants remain user-owned. (c) Save path uses `to_vec_pretty` (deterministic on a given Value tree) for byte-identical idempotency — the NoChange path additionally short-circuits via pre/post Value comparison so we don't rewrite files we didn't semantically alter. (d) Atomic write uses `<path>.tmp` in SAME directory → rename; cross-device (EXDEV) failures on overlayfs/tmpfs setups are the real-world failure mode to avoid.
- T-019 CC_HOOK_BYTES STUB: T-008a shipped `CC_HOOK_BYTES = &[]`. `install_cc_hook_at` short-circuits with `InstallOutcome::StubEmpty { path }` + a helpful WARN when bytes is empty. Overrides default to `default_settings_path()` + `cc_hook_install_path()`.
