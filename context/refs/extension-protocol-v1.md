# Extension Protocol -- v1.0 (frozen)

> Annotated reference guide for the ark extension runtime-RPC surface.
> Companion to `intent-api-v1.md` (core ops) and `wasm-metadata-v1.md`
> (wasm custom section).
>
> Source of truth: `crates/ark-ext-proto/share/extension-protocol.kdl`
> (generated by `cargo run -p ark-ext-proto --bin gen-extension-spec`
> via facet SHAPE reflection over `crates/ark-ext-proto/src/lib.rs`).

## Overview

The extension protocol defines the RPC surface between ark and its
extensions. Three transports speak the same protocol:

| Transport       | Wire format                  | Use case            |
|-----------------|------------------------------|---------------------|
| Subprocess      | JSON-RPC 2.0 over NDJSON     | Most extensions     |
| Wasm component  | wit-bindgen                  | Sandboxed plugins   |
| Compiled-in     | Trait-object dispatch         | Built-in extensions |

All methods, field names, and error codes below are frozen for the
1.x line. See the version compatibility rules at the end.

## Protocol lifecycle

```
ark -> ext:  initialize          (request)
ext -> ark:  InitializeResponse  (with session_token)
ark -> ext:  initialized         (notification)
        ...  (runtime methods)
ark -> ext:  shutdown            (request)
ark:         close stdin
ark:         SIGTERM (after timeout)
ark:         SIGKILL (after timeout)
```

## Method reference

### Lifecycle

| Method        | Kind         | Description                                      |
|---------------|--------------|--------------------------------------------------|
| `initialize`  | request      | Handshake: ark sends protocol version + capabilities; extension responds with its own version, capabilities, and extension info. Host overwrites `session_token` on the response. |
| `initialized` | notification | Extension confirms post-initialize setup is done. |
| `shutdown`    | request      | Graceful teardown. Ark follows shutdown -> stdin close -> SIGTERM -> SIGKILL. |
| `ping`        | request      | Liveness probe. Empty request and response.       |

#### `initialize` params

| Field                  | Type   | Req | Description                                  |
|------------------------|--------|-----|----------------------------------------------|
| `protocol_version`     | string | yes | Ark's `MAJOR.MINOR` version range.           |
| `client_capabilities`  | string | yes | Opaque JSON capabilities object.             |
| `client_info`          | string | yes | Ark version string (diagnostic only).        |

#### `initialize` result

| Field                    | Type   | Req | Description                                |
|--------------------------|--------|-----|--------------------------------------------|
| `protocol_version`       | string | yes | Extension's `MAJOR.MINOR` version range.   |
| `extension_capabilities` | string | yes | Extension's capabilities object.           |
| `extension_info`         | string | yes | `{ name, version }` descriptor.            |
| `session_token`          | string | yes | Host-minted token for reverse-requests.    |

### Async and cancellation

| Method        | Kind         | Description                                       |
|---------------|--------------|---------------------------------------------------|
| `$/cancel`    | notification | Cancel an in-flight request by its JSON-RPC id.   |
| `$/progress`  | notification | Extension emits progress update (token + value).  |
| `task/create`  | request     | Start a long-running op; returns a `TaskId` handle. |
| `task/get`     | request     | Poll task state: pending/running/succeeded/failed/cancelled. |
| `task/cancel`  | request     | Cooperative cancel on a running task.              |

### Event bus

| Method              | Kind         | Description                                    |
|---------------------|--------------|------------------------------------------------|
| `event/subscribe`   | request      | Subscribe to events matching a selector glob.  |
| `event/unsubscribe` | request      | Revoke a prior subscription by id.             |
| `event/emit`        | request      | Extension publishes an event onto ark's bus. Namespace must be `<ext-name>.*` (R11); `ark.core.*` writes are rejected. |
| `event/notify`      | notification | Host delivers a subscribed event to the extension. |

### Intents

| Method              | Kind    | Description                                       |
|---------------------|---------|---------------------------------------------------|
| `intent/register`   | request | Extension advertises a named intent with args schema. Namespace: `<ext-name>.<intent>` (R10). |
| `intent/unregister` | request | Drop a prior intent registration.                  |
| `intent/dispatch`   | request | Ark asks the extension to execute a registered intent. Args are schema-validated before delivery. |

### UI: keybind and status

| Method                  | Kind         | Description                                |
|-------------------------|--------------|--------------------------------------------|
| `ui/keybind/register`   | request      | Advertise a command ID (does NOT bind raw keys -- the user's scene binds keys to the command ID via R16). |
| `ui/keybind/unregister` | request      | Drop a command registration. Extensions MUST unregister in `shutdown`. |
| `ui/status/push`        | notification | Update the status line. Empty text clears. |

### UI: panes

| Method           | Kind    | Description                                         |
|------------------|---------|-----------------------------------------------------|
| `ui/pane/request`| request | Fill a user-declared pane slot or open an ephemeral overlay. |
| `ui/pane/close`  | request | Close a pane previously requested by this extension. |

### Workspace

| Method                         | Kind         | Description                              |
|--------------------------------|--------------|------------------------------------------|
| `workspace/applyEdit`          | request      | Apply text edits (LSP `WorkspaceEdit` shape). |
| `workspace/configuration`      | request      | Read a config value scoped to `<ext-name>.*`. |
| `workspace/showDocument`       | request      | Open a file or URL for the user.          |
| `workspace/showMessage`        | notification | Emit a toast/log line.                    |
| `workspace/showMessageRequest` | request      | Show a message with action buttons; await user choice. |

### Scene

| Method          | Kind    | Description                                         |
|-----------------|---------|-----------------------------------------------------|
| `scene/getRoot` | request | Query the currently-loaded scene path and session CWD. |

### Host syscalls (wasm-only)

These methods are only available to wasm-component extensions.
Subprocess extensions MUST use OS syscalls directly; calling these
from a subprocess returns `ExtensionError::CapabilityDenied`.

| Method           | Kind    | Description                                   |
|------------------|---------|-----------------------------------------------|
| `host/fs/read`   | request | Read a file via the host (UTF-8 only).        |
| `host/fs/write`  | request | Write a file via the host.                    |
| `host/proc/spawn`| request | Spawn a subprocess via the host.              |
| `host/net/fetch` | request | Perform an HTTP request via the host.         |

### Logging

| Method         | Kind         | Description                                   |
|----------------|--------------|-----------------------------------------------|
| `log/write`    | notification | Extension writes a structured log line.       |
| `log/setLevel` | request      | Ark sets the minimum severity for log/write.  |

## Key types

### `LogLevel` (enum)

Variants: `Error`, `Warn`, `Info`, `Debug`, `Trace`. Aligned with LSP
`window/logMessage` types and Rust `tracing::Level`.

### `TaskId` (struct)

Single field `value: string` -- opaque, extension-minted identifier
unique within the session.

## Error codes

All error codes are JSON-RPC 2.0 error objects. Extension-specific
codes use the `ext/` prefix:

| Code                      | Meaning                                       |
|---------------------------|-----------------------------------------------|
| `ext/reserved-namespace`  | Extension tried to register in `ark.core.*`.  |
| `ext/capability-denied`   | Method not available for this transport.       |
| `ext/invalid-params`      | Request params failed schema validation.       |

## Version compatibility contract

Mirrors the intent-API contract. For the 1.x line:

1. **New methods** may be added in a MINOR bump. Extensions MUST
   ignore unknown methods (return `MethodNotFound` per JSON-RPC).
2. **New optional fields** may be added to request/response structs
   in a MINOR bump. Receivers MUST ignore unknown fields.
3. **Removing or renaming** a method or field is a MAJOR change.
4. **Changing a field type** or required/optional status is MAJOR.
5. **Method kind** (request vs notification) is frozen per-method.
6. **Error code strings** may grow (MINOR); changing or removing is MAJOR.
