### 2. Stack::spawn_pane live RPC wiring
**Source:** `crates/ark-view/src/typed.rs:252` — `PaneAttrs` is empty struct in v0.1.
**Flagged by:** claude-code-ext Tier 6b T-038 (ledger row in impl-claude-code-ext.md).
**Symptom:** `ClaudeCodeView` fan-out on `claude-code.subagent.start` computes `ClaudeCodeSubagentAttrs` correctly but cannot call `subagents.spawn_pane(attrs)` with meaningful payload — `PaneAttrs` accepts no fields.
**Fix:** Widen `PaneAttrs` to carry per-view fields (either `serde_json::Value` or generic over `V::Attrs`). Wire `Stack::spawn_pane` → `stack/spawn_pane` RPC → supervisor → ext. Subagent registry auto-wire (item #3) cross-cuts this.
**Effort:** L (2-3 days). Kit needed.

## Non-blocking (nice-to-have v0.2)

### 3. SubagentRegistry auto-wire into accept loop
**Source:** `extensions/claude-code/src/view.rs` — `SubagentRegistry` is public + tested.
**Flagged by:** claude-code-ext Tier 6b T-037 (ledger).
**Symptom:** Registry exists but `on_session_start`'s accept loop doesn't call it. Subagent state updates don't flow through registry → RenamePane emission stays stubbed.
**Fix:** Add `Arc<SubagentRegistry>` field on ClaudeCodeExtension alongside `event_sink`. Call `registry.apply_event(&ext_event)` in the accept loop after bus forwarding.
**Effort:** S. Pairs naturally with item #2.

### 4. CLI `ark ext <name> <verb>` dispatcher
**Source:** `extensions/claude-code/src/lib.rs` — `control_verbs` + `run_install_hooks_verb` + `run_reinstall_hook_binary_verb` + `reload`.
**Flagged by:** claude-code-ext Tier 4 (T-022/T-023) + Tier 8 T-046 (ledger).
**Symptom:** Supervisor collects `ext.control_verbs.v1` advertisements (phase-2 T-029 infrastructure) but no CLI entry point dispatches `ark ext <name> <verb>` to the supervisor → extension RPC.
**Fix:** Add `crates/cli/src/commands/ext_verb.rs` subcommand. Looks up extension by name via running session's status.json. Sends `ControlVerbInvoke { ext, verb, args }` over control socket. Supervisor routes to the extension's `control_verb_invoke` RPC.
**Effort:** M.

### 5. ext_state persistence for list columns
**Source:** `extensions/claude-code/src/columns.rs` — `list_state: Arc<Mutex<CcListColumnState>>` is in-process only.
**Flagged by:** claude-code-ext Tier 7 T-044 (ledger).
**Symptom:** `cc model` / `cc tokens` / `cc cost` visible only within the running ext process. Cross-session `ark list` misses these columns.
**Fix:** Supervisor exposes a pub API `SessionStatus.ext_state[<ext>]: serde_json::Value`. Extension `list_columns` writes its state via this API; `ark list` reads back from persisted `status.json`.
**Effort:** M. Cross-crate (supervisor pub API widening).

### 6. Full-zellij PTY harness for claude-code-ext
**Source:** `extensions/claude-code/tests/claude_code_smoke.rs` — T-048 runs reduced harness (direct socket + NDJSON, no real zellij launch).
**Flagged by:** claude-code-ext Tier 8 T-048 (ledger).
**Symptom:** E2E gate covers wire + state; does not cover actual ark-under-zellij launch + view rendering.
**Fix:** New crate `crates/ark-test-harness` spawns `ark run` with mock-claude-cc symlinked as `claude`; fixtures drive actual subagent burst + assert zellij's view grid via `screencap` or similar.
**Effort:** L (3+ days). Kit needed.

### 7. cc-hook real `include_bytes!` embedding
**Source:** `crates/cli/build.rs` + `extensions/claude-code/src/lib.rs` — `CC_HOOK_BYTES: &[u8] = &[]` (stub).
**Flagged by:** claude-code-ext Tier 1 T-008a (ledger).
**Symptom:** `ark ext claude-code reinstall-hook-binary` emits a helpful WARN but can't extract bytes (stub empty).
**Fix:** Not via cargo-inside-cargo build.rs (F-709 deadlock). Use cargo-dist or a packaging-time step that copies the built `cc-hook` binary into the `ark` binary's release bundle via resource-compilation (OS-specific). Or: sidestep the embedding entirely — `doctor --fix` runs `cargo install --bin cc-hook --path extensions/claude-code`.
**Effort:** M (design-heavy). Kit needed.
